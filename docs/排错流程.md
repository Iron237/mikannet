# Mikannet 全面排错流程

> 结合本项目架构与已踩过的 bug 类别沉淀的排错/审计流程。改完代码 → `docker compose up -d --build` 重建。

## 0. 原则与环境陷阱
- **单写者 SQLite**:`journal_mode=DELETE`(WAL 在 Win Docker bind mount 不可用)、`PRAGMA foreign_keys=ON`、`synchronous=FULL`、`busy_timeout=30000`。三个并发写者(tracker / postprocess worker / APScheduler)都各开 `db_session`。
- **失败不阻塞主链路**:元数据 / ffprobe / 整理 / 推送失败都留痕可重试,不回滚主状态。排错先看 `docker logs mikannet` 的 Traceback。
- **CJK 编码陷阱**:PowerShell/Bash 经 curl/heredoc 把中文按 GBK 编码 → 请求体损坏。涉及中文一律用 Write 写 UTF-8 `.py` + `docker cp` + `docker exec -e PYTHONPATH=/opt/mikannet/backend python`;脚本里 `sys.stdout.reconfigure(encoding="utf-8")`。
- **代理**:外部服务走 `127.0.0.1:10808`;本地服务(qB)直连。宿主 Python(非容器)经代理会 `CERTIFICATE_VERIFY_FAILED`,容器内正常。

## 1. 静态健康(秒级)
- 导入:`docker exec -e PYTHONPATH=/opt/mikannet/backend mikannet python -c "import app.main"`
- 单测:`PYTHONPATH=$(pwd) .venv/Scripts/python -m pytest -q`(标题语料回归 + 状态机用例)
- 前端构建:`npm run build`(Vue 模板编译错误)
- 启动日志:`docker logs mikannet` —— 迁移(`_migrate_columns`)、下载器连接、调度器三 job、postprocess/tracker worker 均无异常

## 2. 数据完整性(只读 DB 审计脚本,见 `scripts/审计示例`)
- **悬挂外键 / 孤儿**:`video_file.torrent_id`、`torrent_episode`、`bd_extra.bd_release_id`、`bd_release.bangumi_id`、`subscription.bangumi_id`、`episode.bangumi_id`、`torrent.subscription_id` 指向不存在的父。
- **is_active 不变量**:每个 episode 至多 1 个 `is_active` 的 VideoFile(`_apply_version_switch` 应保证)。查 >1 的违规。
- **kind 守卫**:`kind != tv`(movie/ova)的番剧不应有带 active 文件的 `regular` 集(剧场版误归第 1 话回归)。
- **计数封顶**:`eps_downloaded ≤ eps_total`(跨季连续编号 / 幽灵集导致超额)。
- **唯一性**:`torrent.guid` / `torrent.info_hash` 无重复(UNIQUE)。
- **容器伪订阅**:`local`/`auto` 子组的合成种子应无 `info_hash`。
- **路径安全**:`video_file.relative_path` 不含 `..` / 不是绝对路径(删盘目录穿越前置)。

## 3. API 冒烟(全部 GET 应 200)
`/api/system/health`、`/api/bangumi`、`/api/bangumi/{id}`、`/api/subscriptions`(全来源)、`/api/bd/releases`、`/api/launch/status`、`/api/config`、`/api/files/unassigned`、`/api/bangumi/calendar/week`、`/api/notifications`、各 `*/status`。

## 4. 业务不变量(运行期)
- **状态机 ↔ qB 对账**:DOWNLOADING / COMPLETED / DOWNLOAD_ERROR 与 qB 实际一致;坏种 / 无进度计时复位;ERROR 自愈仅在 qB 恢复活跃态时回 DOWNLOADING(暂停态不打回)。
- **智能下载选种**:同字幕组锚定 + 去重——无重复集、无死循环、缺口留待。
- **整理**:撞名跳过、`commit` 早于 `organize`、仅 qB 后端、仅正片+带号特别篇。
- **删除**:带「已绑 BD 发行」的番剧可删(FK 分阶段 flush);勾删文件对本地/容器(无 info_hash)直接删盘且限下载根内。

## 5. 新功能专项
- **协议头**:路径映射(盘符 / CJK / 空格 / 末尾斜杠)、`launch_url` URL 编码 ↔ ps1 正则解析一致、令牌校验、白名单前缀**不能被子串绕过**(`Z:\番剧` 不应放行 `Z:\番剧2`)、`.bat`/`ps1` 转义、未配置根 → 按钮 null 安全。
- **删盘安全**:`os.remove` 解析后须仍在下载根内,绝不碰已购原盘 / 做种中文件。
- **订阅全来源**:徽章 + 操作限制;批量删除 / 全选 / 向导不被破坏;`local`/`auto` 行不渲染启停 / 规则 / 健康灯。
- **BD 碟枚举**:`_owned_discs` 读挂载的磁盘 I/O 异常安全;多碟 / 单碟根。

## 6. 代码审查(并行 agent,按 bug 类别分维度)
1. 原生启动新功能(路径映射 / 编码一致 / 转义 / 令牌 / 白名单 / 穿越)
2. 删除 / 事务 / FK(UOW 删序、autoflush、session 毒化、删盘穿越)
3. 状态机 / tracker / 后处理(计时复位、is_active 仲裁、drain 幂等、多线程写)
4. 解析 / 映射 / 智能下载(选种死循环/漏集/重复、kind 守卫、解析边界)
5. API / 前端 / 资源(全来源副作用、null 安全、定时器泄漏、v-html、密钥)

## 7. 修复与复验
高置信问题 → 修复 → `pytest` + 审计脚本复跑 → `docker compose up -d --build` → `docker logs` 确认 → 提交(作者 Iron237,Co-Authored-By 行)+ push。
