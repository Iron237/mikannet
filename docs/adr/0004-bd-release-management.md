# BD 发行作为独立实体管理(特典 + 拥有状态门控下载)

## 背景与决策

BD(蓝光)不只是「更高画质的正片」——一套 BD 发行还带大量**特典**:特别动画、放送后短剧、NCOP/NCED、菜单、原声/角色歌 CD(FLAC)、设定图集、扫描书子(JPG)。这些里**非视频内容(音频/图片)**用现有 `VideoFile`(以视频文件 + ffprobe + 剧集映射为中心)根本表达不了,而视频特典若塞进剧集网格当 `OTHER` 集又会污染正片视图。用户还要按「套」管理收藏:每套 BD 标记**已购买/未购买**,并据此决定要不要再去网络下 BDRip。

真实数据有两种形态,差异巨大:
- **压制组 BDRip**(如 `[DBD-Raws]…[BDRip]`):文件名规整,特典分门别类放在 `CDs/`、`图集/`、`特别动画/`、`短剧/`、`menu/`、`Scans/` 子目录,**可按目录名+文件名直接归类**。
- **自购原盘 MAKEMKV 翻录**(如 `MAIDRAGON S/DISC1/BDMV/STREAM/00000.m2ts`):未压制原盘,数字命名 m2ts(单碟 7GB×多、3 碟),哪个是正片/特典只能靠 `PLAYLIST/*.mpls` 或大小猜,**极不可靠**。

**决定**:把 **BD 发行(BdRelease)作为番剧下的独立实体**建模,与剧集/VideoFile 解耦。

- 新表 `bd_release`(归属番剧、来源形态 bdrip/raw_disc、根路径、拥有状态 owned、碟数/总大小)+ `bd_extra`(特典条目,`media_kind` ∈ video/audio/image,专门承载 FLAC/JPG)。**正片不进 bd_extra**(仍走剧集网格),BD 特典也不再混进网格当 OTHER 集。
- **拥有状态双重用途**:① 收藏管理(BD 总览页逐套切换);② 下载门控——番剧一旦视为「已购买(有原盘)」就**完全排除出自动下载**(auto-best + RSS 轮询都跳过),纯本地/原盘管理。用 `Bangumi.bd_owned` 表达番剧级门控。
- **两类来源都扫**:下载根里 `片源=BD` 的合集 → `bdrip` + `owned=false`;额外配置的「已购原盘目录」(独立 CIFS 挂载)→ `raw_disc` + `owned=true`。
- **原盘 v1 仅粗粒度占位**:`raw_disc` 只记路径/碟数/总大小/拥有,**不拆 m2ts**。深度解析(mpls/大小 → 正片/NC/特典)留待后续——原盘归类不可靠,且正片识别本就以另下/另压的版本为准。

## 考虑过的替代

- **复用 Episode/VideoFile 表达特典**:VideoFile 以视频+ffprobe+集号为中心,装不下 FLAC/JPG;视频特典当 OTHER 集会污染正片网格。否决——特典是「按套收藏」的浏览对象,不是「按集追番」的对象,语义不同。
- **v1 就深度解析 MAKEMKV 原盘**:解析 mpls + m2ts 大小拆正片/特典。否决——不同压制/碟编号规律不一、易错;占位已满足「记录拥有 + 收藏」的核心诉求。保留为后续增量。
- **拥有状态只做标签、不门控下载**:用户明确要「已购买→完全不自动下载」,故拥有状态参与 auto-best/RSS 跳过。

## 后果

- 新增只读串流端点把 NAS 上的 BD 特典(图片/音频)喂给前端(画廊/播放),文件仍不动(守 ADR-0001)。
- 「已购原盘目录」要新增一个 CIFS 挂载(像 `/import-nas`);路径可配,缺失则该来源跳过、不影响其余。
- 番剧详情页新增「BD/特典」区,番剧库新增「BD 收藏」总览页。
- 死种另加「无进度→暂停」温和档(与现有「0 做种→删除换源」并存),属独立小改,不在本 ADR 范围。
