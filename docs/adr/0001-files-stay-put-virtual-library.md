# 下载文件保持原样,番剧库为虚拟视图

> **⚠️ 已被 [ADR-0002](0002-organize-via-inplace-rename.md) 修订(2026-06-13)。**
> 不再坚持"绝不重命名":改为用 qB `renameFile` **原地重命名**整理成 Jellyfin 结构(保种、不复制、不硬链接),可开关。
> 本文档的"虚拟库""标题解析是核心""不移动/不复制"原则仍然成立;变化的只是"允许原地改名 + 写 NFO 落盘",详见 ADR-0002。

下载完成的视频文件保持字幕组原始命名留在 qBittorrent 下载目录中持续做种,系统不重命名、不移动、不硬链接;WebUI 的番剧库完全由数据库中「文件 ↔ 番剧/剧集」映射关系渲染,映射由种子标题解析建立。

原因:(1) 当前架构是 Win PC 上的 Docker 通过 SMB 写入 NAS,跨 SMB 创建硬链接不可行,AutoBangumi 式硬链接整理走不通;(2) 移动/重命名会中断做种,复制会翻倍占用空间;(3) 用户不依赖 Jellyfin/Plex,展示需求全部由本系统 WebUI 满足。

## Consequences

- 种子标题解析(标题 → 番剧 + 集数)成为核心组件,库视图、推送文案、去重都依赖它。
- 「整理模块」(硬链接出规整目录)作为可选功能预留,待系统迁移到 NAS 本机 Docker(同文件系统)后才可启用。
